# 2026-05-11 — Discord Footer & Channel Dispatch Debugging

## 重大收获

### Discord 回复不显示问题的完整解决

**四大连环问题：**

1. **模型 rate limit** — `openai-codex/gpt-5.4` Codex API 限速，model_call 卡死
   - 解决：`channels.modelByChannel` 把 Discord 频道改为 `deepseek/deepseek-v4-flash`

2. **Footer 补丁导致 ReferenceError** — `channel` 变量声明在 `if` 块内但在块外被引用
   - 解决：手写 5 处精准 edit，加 try-catch 保护

3. **Group chat visibleReplies 配置错误** — 设为 `"message_tool"` 导致只有 message tool 发的才可见
   - 解决：改为 `true`

4. **Gateway 重启被 SIGTERM 截断** — `openclaw gateway restart` 发 SIGTERM 会杀当前 shell
   - 解决：用 `openclaw gateway start >/dev/null 2>&1 & disown` 后台启动

### Footer 补丁关键设计

- `formatDiscordFooterLine` 用 Unicode emoji（\u{1F9E0}, \u23F1, \u{1F4C2}）
- Discord 分隔线：`────────`（8个 dash，有换行前后）
- 所有 footer 代码包 try-catch，失败不影响消息发送
- `responseUsageMode` 默认保持 `"off"`，用户手动 `/usage tokens` 开启
- `channel` 变量在 `if` 块外部声明，避免 ReferenceError
